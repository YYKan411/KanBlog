# 解碼室學習路徑（完整自學素材）

呢份係配合 Lab 工具嘅學習地圖。建議：**每關先自己寫 → 執行 → 先睇結果行數 → 先問自己錯邊 → 先要提示**。

---

## 你點樣先唔會俾 engineer 呃到

1. **先問粒度**：呢條 query 係「每人一行」定「每單一行」？JOIN 之後行數暴升通常係重複計。
2. **先問邊界**：NULL、取消單、時區、重複鍵——結果包唔包？
3. **先問時間窗**：`>=` 仲係 `>`？包唔包今日？
4. **先問度量定義**：GMV 包唔包退款？「活躍」係登入定交易？
5. **先要求 EXPLAIN**：大表全表 SCAN + SELECT * = 紅旗。

---

## Stage 0→1　語法肌肉記憶

| 關卡 | 你要內化嘅句子 |
|------|----------------|
| SELECT | 資料喺表，拎出嚟先睇到 |
| WHERE | 過濾列，唔係過濾欄 |
| ORDER BY / LIMIT | 排序同 Top N 係報表日常 |
| GROUP BY | 聚合前要講明「按邊個組」 |

練習口訣：`SELECT 欄 → FROM 表 → WHERE 列 → GROUP BY 組 → HAVING 組過濾 → ORDER BY → LIMIT`

---

## Stage 1→10　關係同視窗

| 關卡 | 常見坑 |
|------|--------|
| INNER JOIN | 冇配對就消失——樣本偏差 |
| LEFT JOIN | 右邊 NULL 係訊號，唔係 bug |
| RANK() | 同 GROUP BY 唔同：行數唔減 |
| 累計 SUM OVER | 要有明確時間序 |

口訣：JOIN 決定「邊啲實體仲喺場」；Window 決定「喺場內點計奔跑值」。

---

## Stage 10→50　髒數據實戰

現實數據會有：

- `NULL`（未知）vs `''`（空字串）vs `0`
- 日期：`2024-10-01` / `01/10/2024` / `Oct 1, 2024`
- 重複鍵、大小寫、前後空白
- 負數庫存、評分 > 5

Analyst 流程：**偵察（COUNT / IS NULL / HAVING）→ 定清潔規則 → 先寫防禦性 WHERE → 先聚合**。

---

## Stage 50→100　效能同架構

| 概念 | 判斷標準 |
|------|----------|
| EXPLAIN QUERY PLAN | `SCAN` vs `SEARCH USING INDEX` |
| INDEX | 建喺高頻 WHERE / JOIN 欄；唔好亂建 |
| 投影 | 禁止無腦 `SELECT *` |
| 隱式 JOIN | `FROM a, b` 高危，改 `JOIN ... ON` |

想像數據量 ×1000 萬——你嘅寫法仲站唔站得住？
